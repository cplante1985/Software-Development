# Software-Development

•	How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?

The best way to interpret user needs and priorities is to simply ask the users and stakeholders. The users are going to be using our program once completed so their wants must take center stage. The Product Owner’s job is to communicate with the users to gain feedback and ideas for the software. User stories are used to commune with the developers to add those changes into the final project.

•	How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

I plan to use two Agile values while developing programs, customer collaboration over contract negotiation and responding to change over following a strict plan. I want to be able to quickly redirect my focus when changes appear within my projects due to the user’s feedback. I also want to constantly reinforce the importance of the customer’s priorities and ideas. 


•	What does it mean to be a good team member in software development?

I believe to be a good team member in software development, you must be able to collaborate with others and willing to learn from others. A good developer should always consider a different approach to software, such as learn a different program or a different model for a project.
